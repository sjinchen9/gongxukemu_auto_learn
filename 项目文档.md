# 重庆人社局在线学习自动化工具

## 项目概述

针对重庆人社局在线培训平台的自动化学习工具。通过屏幕截图 + NCC模板匹配 + pyautogui 模拟点击，完全本地运行。

---

## 当前版本：v3.2（2026-06-04）

### 架构：双状态 + 全局弹窗 + 链式处理

```
状态机只有 2 个状态：VIDEO / COURSE
弹窗确认：全局优先检测，不进入状态机
点击关闭后：链式处理弹窗确认（同一轮内完成）
```

### 状态判定（多信号综合）

| 信号 | 含义 |
|------|------|
| 关闭按钮 | 视频页特有 |
| 下一节按钮 | 视频播放中 |
| 大面积暗色 | 视频播放器（播完黑屏） |
| 灰色+蓝色标签共存 | 课程列表 |

3 个视频页信号命中任意 1 个 → VIDEO，否则 COURSE。

### 流程

```
每轮扫描：
  1. 全局弹窗检测 → 有弹窗立即点确认
  2. 自检页面类型 → VIDEO / COURSE
  3. VIDEO → 找"下一节" / 黑屏→立即点关闭 / 超时也点关闭
     点击关闭后 → 链式找弹窗确认按钮点之
  4. COURSE → 找"未完成" / 滚动 / 切选项卡
  5. 验证失败 → 重置状态，下轮重新自检
```

### 检测流水线

```
全屏截图 → 颜色预筛选 → NCC模板匹配 → 颜色二次验证（蓝色≥10%）→ 点击
```

### 关键设计决策

**为什么只有 2 个状态？**
- "视频正在播放"和"全部小节学完"是同一个页面——都有关闭按钮
- 区分方式：暗色区域 > 50% = 播完了 → 立即点关闭
- ALL_DONE 和 CONFIRM 作为独立状态增加了复杂度但没增加价值

**为什么弹窗不进入状态机？**
- 弹窗可以在任何时候出现
- 全局优先检测确保弹窗不被状态切换延迟

**为什么关闭后链式处理弹窗？**
- 关闭→弹窗是必然的因果链
- 等下一轮再处理可能导致弹窗超时消失

## 使用说明

### 环境

- Windows 10/11 + Python 3.7+
- `pip install pyautogui Pillow numpy`

### 运行

```bash
python auto_learn.py
```

| 按键 | 功能 |
|------|------|
| `s` | 手动模式（每秒扫描） |
| `a` | 自动模式 |
| `q` | 退出 |

### 扫描间隔

- 前 3 轮：15 秒
- VIDEO 状态：60 秒
- COURSE 状态：10 秒

---

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| SCAN_VIDEO | 60s | 视频页扫描间隔 |
| SCAN_OTHER | 10s | 课程页扫描间隔 |
| SCAN_INITIAL | 15s | 前3轮快速扫描 |
| MAX_CLICKS | 50 | 最大点击次数 |
| TEMPLATE_SCORE | 0.45 | NCC匹配阈值 |
| TEMPLATE_STEP | 8px | 搜索步长 |
| MISS_THRESHOLD | 3 | 连续未检测到阈值 |
| SCROLL_LIMIT | 5 | 课程页滚动上限 |

---

## 已知限制

- 依赖模板与页面匹配，分辨率/主题变化可能需重新截模板
- 仅支持单显示器
- `find_incomplete` 颜色检测在复杂页面可能误匹配
- 关闭按钮模板需与实际页面匹配
