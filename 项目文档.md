# 重庆人社局在线学习自动化工具

## 项目概述

针对重庆人社局在线培训平台的自动化学习工具。通过屏幕截图 + OCR文字识别/ NCC模板匹配 + pyautogui 模拟点击，完全本地运行。

---

## 当前版本：v4.0 OCR驱动版（2026-06-06）

### 架构：双状态 + 全局弹窗 + 链式处理

```
状态机：VIDEO / COURSE 两个状态
弹窗确认：全局优先检测，不进入状态机
关闭→弹窗：链式处理（同一轮内完成）
```

### v4.0 vs v3.x 对比

| 方面 | v3.x | v4.0 |
|------|------|------|
| 识别方式 | NCC模板匹配 + 颜色检测 | PaddleOCR文字识别为主，NCC翻页栏保留 |
| 模板文件 | 9个png模板 | 1个（仅翻页栏） |
| "未完成"vs"已完成" | 颜色聚类+逐字NCC（常误判） | OCR精确匹配exact=True（准确） |
| 弹窗检测 | NCC+蓝色验证+结构上下文 | OCR搜"确定/确认"+蓝色验证+结构上下文 |
| 翻页栏 | ✅ 稳定 | 从v3搬回NCC方案 |
| "下一节"检测 | NCC | OCR精确匹配 |
| "关闭"检测 | NCC（频繁点歪） | OCR精确匹配（稳定） |
| 选项卡（必修/选修） | NCC模板匹配 | OCR精确匹配 |
| 点击验证 | 1.5s后截图验证 | 同v3 |
| has_page() | 颜色检测 | OCR关键词+颜色兜底 |
| Python版本 | 3.13 | 3.11（PaddleOCR要求） |

### 核心改进：OCR + NCC 混合方案

**教训**：v4初期全盘推倒v3，导致大量回归bug。最终方案是：保留v3全部架构，只替换识别层中不可靠的部分。

**最终方案**：
- **OCR**：文字识别（"未完成"/"已完成"/"确定"/"下一节"/"关闭"/选项卡）
- **NCC模板匹配**：仅翻页栏（从v3搬回，已验证可靠）
- **颜色检测**：弹窗上下文验证（白色对话框+遮罩）、has_page()兜底

## 历史版本：v3.2（2026-06-05）

### 架构：双状态 + 全局弹窗 + 链式处理

```
状态机：VIDEO / COURSE 两个状态
弹窗确认：全局优先检测，不进入状态机
关闭→弹窗：链式处理（同一轮内完成）

每轮扫描：
  1. 截屏 → 页面检测
  2. 全局弹窗检测 → 有弹窗立即点确认
  3. 自检页面类型（有关闭/下一节/暗色→VIDEO，否则→COURSE）
  4. 按状态处理：
     VIDEO → 找下一节 → 黑屏则点关闭 → 链式点击弹窗
     COURSE → 找未完成 → End跳底 → 找翻页 → Home回顶找选项卡
  5. 点击后验证 → 失败则重置状态
```

### 检测流水线

```
全屏截图 → 颜色预筛选 → NCC模板匹配（阈值0.45）→ 蓝色二次验证（≥10%）→ 点击
```

## 使用说明

### 环境

- Windows 10/11 + Python 3.7+
- `pip install pyautogui Pillow numpy`

### 运行

```bash
python auto_learn.py
```

| 按键 | 功能 |
|------|------|
| `s` | 手动模式（每秒扫描） |
| `a` | 自动模式 |
| `q` | 退出 |

### 扫描间隔

- 前3轮：15秒
- VIDEO：60秒
- COURSE：10秒

---

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| SCAN_VIDEO | 60s | 视频页扫描间隔 |
| SCAN_OTHER | 10s | 课程页扫描间隔 |
| SCAN_INITIAL | 15s | 前3轮快速扫描 |
| MAX_CLICKS | 50 | 最大点击次数 |
| TEMPLATE_SCORE | 0.45 | NCC阈值 |
| TEMPLATE_STEP | 8px | 搜索步长 |
| MISS_THRESHOLD | 3 | 连续未检测到阈值 |
| SCROLL_LIMIT | 5 | 滚动上限 |

## 已知限制

- 依赖模板与页面匹配，分辨率/主题变化需重新截模板
- 仅支持单显示器
- find_incomplete 颜色检测在复杂页面可能误匹配
